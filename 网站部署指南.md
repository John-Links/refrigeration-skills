# 极简网站部署指南（GitHub Pages）

## 老板你看到的成果
本仓库自动部署一个**静态网站**到 GitHub Pages。地址：
```
https://john-links.github.io/refrigeration-skills/
```

## 类比
- GitHub Pages = GitHub 免费送的"小网站空间"
- 只要把 HTML 文件推上去，GitHub 自动帮你"上线"
- 不需要买服务器、不需要域名（虽然域名以后可以买）

## 怎么启用（只需 1 次）

### 第 1 步：打开仓库的 Settings 页面
访问：https://github.com/John-Links/refrigeration-skills/settings/pages

### 第 2 步：选 Source = "GitHub Actions"
- 不要选 "Deploy from a branch"
- 选 **"GitHub Actions"**
- 这样我们刚才写的 `.github/workflows/pages.yml` 才会生效

### 第 3 步：保存
- 点 "Save"
- 会跳回页面，看到 "Your site is live at..." 大约 1-2 分钟后

### 第 4 步：等老板推完代码
- 因为这是 GitHub Actions 部署，需要先 push 到 GitHub
- 老板粘 `git push` 之后，GitHub 自动：
  1. 跑测试（test.yml）
  2. 部署网站（pages.yml）
  3. 网站自动上线

## 完整时间线
```
1. 老板 git push
   ↓
2. GitHub 检测到 push
   ↓ (10 秒)
3. test.yml 跑测试（4 个 Python 版本）
   ↓ (1-2 分钟)
4. pages.yml 部署网站
   ↓ (30 秒)
5. 网站上线：https://john-links.github.io/refrigeration-skills/
```

## 网站有什么
- ✅ Skills 介绍（7 个）
- ✅ 用户群体（4 类人）
- ✅ 快速开始（3 步）
- ✅ Star / Fork 按钮
- ❌ 没有支付功能（暂时不做，复杂）
- ❌ 没有登录注册（暂时不做）

## 为什么不一步到位做"完整网站"
老板说过"极简 A 形态"先跑起来：
- 极简站 = 1 周搞定，0 成本
- 完整站（带会员、支付、登录）= 1-3 个月 + ICP 备案 + 1-3 万成本

先把 Skills 跑起来，把流量先积累起来，再决定要不要做"完整站"。

## 故障排查
如果网站 404：
1. 检查 Settings → Pages → Source 是不是 "GitHub Actions"
2. 检查 Actions 标签页：https://github.com/John-Links/refrigeration-skills/actions
3. 看 pages.yml 是不是跑成功
4. 等 5 分钟（DNS 缓存）

如果网站样式乱了：
- index.html 已用 inline CSS，不需要外部资源
- 直接 commit 改 CSS 就好
